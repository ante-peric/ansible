3. Uvod U Modules Basic And Add Hoc Naredbe

Module – je naredba koju Ansible može pokrenuti. Lista modula se izlistava s naredbom:

Authorative documentation

$ ansible-doc

Lista Modula

$ ansible-doc -l

Službena stranica za Module - Index Module:

https://docs.ansible.com/ansible/2.9/modules/modules_by_category.html

Za informacije o modulu koristimo sljedeću naredbu:

$ ansible-doc module_name
example:
$ ansible-doc user

Napomena: Ako provjerimo python skriptu za ova modul i kad vidimo “=” to znaci da je ta opcija madantory, sve ostalo je optional.

Stanja Modula:

stableinterface - Modul je stabilan i siguran za uporabu
preview - Modul je “tech preview” i keywords i argumenti se mogu promjenuti u budućnosti
deprecated - Modul se nebi smio više koristiti i u budućnosti će biti ukljonjen
removed - Modul je uklonjen iz uporabei dokumentacija još uvijek postoji za pomoć korisnicima kod migracije za njegovu zamjenu

Module support status:

suported_by polje u ansible-doc ukazuje tko je odhovoran za potporu modula

core - Odgovoran je Core Ansible Developers Team
curated - Primarna odgovornost je partnerski tim i tvrtke. Izmjene su nadgledane od strane Core Ansible Team
community - Potpora je u potpunosti od strane zajednice

raw Module:

raw Modul se koristi na održavanje (pristup) udaljenom računalu koji još nije postavljen od strane Ansible (nema SSH key, i nema instaliran Python3)

Primjer - Instalacija python3 paketa na ansible-nesvjesnu radnu stanicu

# ansible -u root -i inventory a3a.8.5.caksonic.local --ask-pass -m raw 'yum -y install python3'

Add Hoc Naredba

Syntax:

$ ansible host_pattern [options]
$ ansible hosts -m module [-a ‘module arguments’] [-i inventory]

• hosts – identificira udaljena računala
• -m module označava korišteni modul
• module arguments – označava argumente modula
• inventory – identificira inventory file

Primjer ping i service modula:

$ ansible all -m ping
$ ansible all -m service -a “name=httpd state=started”
